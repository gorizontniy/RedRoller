Документация

1. Организация

Откройте Yandex Cloud Console, нажмите название организации в левом верхнем углу и скопируйте идентификатор в поле ID организации.

2. Платёжный аккаунт

Откройте раздел биллинга, выберите нужный платёжный аккаунт и вставьте его ID в поле ID платёжного аккаунта.

3. Сервисное облако

Это облако, где живёт сервисный аккаунт. Скопируйте ID облака в поле ID сервисного облака. Это облако скрипт удалять не должен.

4. Режим крутки

Сначала выберите режим в форме аккаунта. Гибридная крутка скрывает проектные поля и создаёт disposable cloud/folder сама. Крутка 1 проекта раскрывает поля ID целевого облака и ID каталога 1 проекта.

5. JSON-ключ

Откройте сервисный аккаунт, нажмите Создать ключ → Создать авторизованный ключ, затем нажмите Скачать файл с ключами. В Redroller вставляется весь скачанный JSON-файл целиком. Не копируйте отдельно блок Ваш открытый ключ или блок Ваш закрытый ключ: по отдельности они не являются нужным JSON-ключом.

6. Цели и зоны

В форме аккаунта или во вкладке Цели настройте белый список: Целевые CIDR и Целевые IP. В блоке Зоны ролла отметьте зоны, где нужно крутить адреса.

7. Поведение после успеха

Остановиться после первого найденного IP — безопасный режим для первого запуска. Продолжать и собирать несколько IP оставляет успешный адрес, добавляет его cloud/folder в изоляцию и запускает следующий disposable cloud/folder.

8. Изоляция

Во вкладке Изоляция выберите аккаунт и отдельной кнопкой сохраните cloud-id и folder-id, которые нельзя удалять или использовать в ролле. Общая кнопка сохранения аккаунта эти списки не трогает.

9. Профиль задержек

По умолчанию используется moderate_fast: панель ускоряет обычные паузы ролла, но сохраняет ожидания IAM propagation и отдельную quota safety delay.

10. Поддержать проект

Если Redroller экономит время и хочется помочь развитию, можно поддержать проект через DaLink или TON.

Изоляция

Изоляция сохраняется отдельно от формы аккаунта. Выберите аккаунт, укажите cloud-id и folder-id рабочих ресурсов и нажмите отдельную кнопку сохранения.

Разрешены только латинские буквы нижнего регистра, цифры и дефис. Если хотя бы один ID некорректный, список не сохранится.
