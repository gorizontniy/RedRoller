Документация

1. Организация

Откройте Yandex Cloud Console, нажмите название организации в левом верхнем углу и скопируйте идентификатор в поле ID организации.

2. Платёжный аккаунт

Откройте раздел биллинга, выберите нужный платёжный аккаунт и вставьте его ID в поле ID платёжного аккаунта.

3. Сервисное облако

Это облако, где живёт сервисный аккаунт. Скопируйте ID облака в поле ID сервисного облака. Это облако скрипт удалять не должен.

4. Режим крутки

Сначала выберите режим в форме аккаунта. Крутка облаков скрывает проектные поля и создаёт disposable cloud. Крутка 1 проекта раскрывает поля ID целевого облака и Folder ID проекта.

5. JSON-ключ

В сервисном аккаунте создайте авторизованный JSON-ключ и вставьте весь файл целиком в поле JSON-ключ сервисного аккаунта.

6. Цели и зоны

В Целевые CIDR или Целевые IP вставьте нужные диапазоны. В блоке Зоны ролла отметьте зоны, где нужно крутить адреса.

7. Изоляция

Во вкладке Изоляция облака выберите аккаунт и отдельной кнопкой сохраните cloud-id, которые нельзя удалять или переиспользовать. Общая кнопка сохранения аккаунта этот список не трогает.

8. Профиль задержек

По умолчанию используется moderate_fast: панель ускоряет обычные паузы ролла, но сохраняет ожидания IAM propagation и отдельную quota safety delay.

Изоляция облака

Изоляция сохраняется отдельно от формы аккаунта. Выберите аккаунт, укажите cloud-id и нажмите отдельную кнопку сохранения.

Разрешены только латинские буквы нижнего регистра, цифры и дефис. Если хотя бы один ID некорректный, список не сохранится.

Telegram

Уведомления общие для всей панели. При успешном выпадении IP скрипт отправит сообщение в этот чат.

Создайте бота через BotFather, вставьте token, затем узнайте chat_id через getUpdates или специального helper-бота. Если token уже сохранён, поле можно оставить пустым.
